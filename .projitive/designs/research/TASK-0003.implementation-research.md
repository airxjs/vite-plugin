## Implementation Plan

### 1. Example Framework

创建 `examples/minimal/` 最小示例：

```
examples/minimal/
├── package.json         # 依赖 vite^5.0.0, airx^0.3.1
├── vite.config.ts       # 引入 vite-plugin-airx
├── index.html
├── tsconfig.json
└── src/
    ├── main.tsx
    └── App.tsx
```

### 2. GitHub Actions CI/CD Workflow

创建 `.github/workflows/check.yml`：

```yaml
name: Code Check

on:
  push:
    branches: ["main", "dev"]
  pull_request:
    branches: ["*"]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'
      - run: npm ci
      - run: npm run build

  typecheck:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'
      - run: npm ci
      - run: npx tsc --noEmit

  example-test:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'
      - run: npm ci
      - name: Build example
        run: |
          cd examples/minimal
          npm install
          npm run build
```

### 3. Pre-Release Checklist（文档补充）

在 `release-verification-baseline.md` 中补充具体 checklist：

- [ ] `npm run build` 成功
- [ ] `npm run typecheck` 无错误
- [ ] `examples/minimal` 构建成功
- [ ] JSX 产物正确（`__airx__.createElement` 存在）
- [ ] package.json version 已更新
- [ ] CHANGELOG.md 已记录
- [ ] `npm publish --dry-run` 文件列表正确
- [ ] Git tag 已打

### 4. npm scripts 增强

建议在 package.json 新增：

```json
{
  "scripts": {
    "build": "tsc",
    "typecheck": "tsc --noEmit",
    "test:example": "cd examples/minimal && npm install && npm run build"
  }
}
```

### 5. E2E 测试方案

当前插件是构建链路插件，E2E 测试 = 构建 + 检查产物。

测试流程：
```
1. 安装插件（workspace:* 或本地路径）
2. 在空项目中使用插件配置
3. 运行 vite build
4. 检查构建产物中是否正确注入 __airx__.createElement / Fragment
```

## Risks and Open Questions

### Known Risks
1. **workspace 依赖问题**：examples/minimal 使用 `workspace:*` 引用本地插件时，npm install 可能无法正确解析
2. **Vite 7 验证**：主站已在用 Vite 7，但插件尚未在 CI 中验证

### Assumptions
1. 当前插件仅需覆盖构建验证，不需要运行时测试
2. CI 可使用 Ubuntu + Node 20 作为标准环境

### Unresolved Questions
1. **测试框架**：是否需要引入 jest/vitest 做单元测试？还是仅靠构建产物验证？
2. **发布方式**：是否使用 GitHub Actions 自动发布到 npm？还是手动发布？
3. **workspace 版本锁定**：examples/minimal 的 npm install 是否会影响 CI？
