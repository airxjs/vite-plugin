## Code Architecture and Implementation Findings

### source/index.ts#L1-L18
```typescript
import { UserConfig } from 'vite'

export function VitePluginAirx() {
  return {
    name: 'airx',
    config(config: UserConfig, env: { mode: string, command: string }) {
      config.esbuild = {
        ...config.esbuild,
        jsx: 'transform',
        jsxFragment: '__airx__.Fragment',
        jsxFactory: '__airx__.createElement',
        jsxInject: 'import * as __airx__ from \'airx\''
      }
      return config
    },
  }
}
```

**关键点**：
- 通过 `config.esbuild` 注入 JSX 转换配置，这是 Vite 内置 esbuild 的标准方式
- `jsx: 'transform'` 表示使用 esbuild 的 JSX  transform，而非 preserve（保留 JSX）
- `jsxInject` 在每个文件顶部注入 airx runtime 的 import 语句
- **Vite 7 兼容性**：Vite 7 仍然使用 esbuild 作为默认 JSX 处理器，esbuild 配置方式未变
